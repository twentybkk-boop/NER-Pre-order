แนวทางแก้ไขปัญหาข้อความเด้งผิดช่องแชท (ข้อ 1)

เดิมทีถ้าใช้โค้ด liff.sendMessages() ข้อความจะถูกส่งลงในช่องแชทที่ลูกค้ากดลิงค์ขึ้นมาเท่านั้น (เช่น แชทส่วนตัวหรือแชทกลุ่ม) วิธีที่ถูกต้องเพื่อส่งออเดอร์ตรงเข้า **LINE OA ของร้านเสมอ** มีดังนี้:

1

ส่งข้อมูลเข้า Backend

แทนที่จะให้ LIFF ส่งแชทเอง ให้กดส่งข้อมูลออเดอร์พร้อม userId ไปยัง Server ของร้าน

2

ใช้ Messaging API Push

Server ใช้ **LINE Messaging API (Push Message)** ยิง Flex Message เข้าหาลูกค้าใน LINE OA โดยตรง

3

Auto-Open LINE OA

หลังสั่งซื้อสำเร็จ ให้หน้าเว็บ redirect ไปยัง URL Scheme: line://ti/p/@your_line_id เพื่อเปิด LINE OA ทันที

ตัวอย่างโค้ด JavaScript (LIFF Frontend) main.js 
// 1. ดึง Profile เพื่อเอา userId ของลูกค้า
const profile = await liff.getProfile();
const customerUserId = profile.userId;

// 2. ส่งข้อมูลออเดอร์ไปยัง Backend Server ของร้าน
const orderData = {
    userId: customerUserId,
    name: "คุณ ภพ",
    phone: "092-525-2594",
    items: [
        { name: "ชุดจุ่มหมูทะเล", qty: 1, price: 429, note: "ไม่เอาหมู ไม่เอาผัก ไม่เอาทะเล" },
        { name: "โมจิไอศครีม - ชาไทย", qty: 1, price: 39, note: "ไม่เอาไอติม" }
    ],
    totalPrice: 468
};

await fetch('https://your-api.com/api/create-order', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(orderData)
});

// 3. Redirect ลูกค้าเปิดเข้าแชท LINE OA ของร้านโดยอัตโนมัติ
liff.openWindow({
    url: "https://line.me/R/oaMessage/@your_line_oa_id/",
    external: false
});
